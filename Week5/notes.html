Notes

This week i studied about testing and debugging in Javascript. Debugging is the process of testing, finding, and reducing bugs (errors) in computer programs. Errors are caused when something goes wrong in a program, There are different types of errors, and they are usually caused by the following:

  • System errors - there's a problem with the system or external devices with which the program is interacting.
  • Programmer error - the program contains incorrect syntax or faulty logic.
  • User error - a data was entered incorrectly, which the program cant recognise.

It's important to be able to recognize what errors we have so that we can properly test and debug them.

Console.log()

One debugging method is console.log(). Using this allows us to display a value in the console so we can see what value is being calaculated and can see if it's our desired outcome.

Example from w3school a = 5;
b = 6;
c = a + b;
console.log(c);

By doing this we will see that c is equal to 11 and and know whether or not our program is working correctly.

Breakpoints

In the debugger window, you can set breakpoints in the JavaScript code. At each breakpoint, JavaScript will stop executing, and let you examine JavaScript values. After examining values, you can resume the execution of code (typically with a play button).

Debugger Keyword

The debugger keyword is similar to settting a breakpoint. It stops the execution of the program where it is placed before continuing.

var x = 15 * 5;
debugger;
document.getElementById("demo").innerHTML = x;

you can activate debugging in your browser with F12, and select "Console" in the debugger menu.
